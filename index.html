Mindo OTP User Consent

This page describes how users of the Mindo mobile application give consent to receive one-time SMS verification codes from our toll-free number.

How users opt in

Within the Mindo app, users navigate to the authentication screen labeled "Continue with Email or Phone".

On this screen, users manually enter their own mobile phone number into a text field and then tap the "Next" button. By tapping this button, the user is explicitly requesting that we send a one-time verification code (OTP) by SMS to the phone number they provided.

This action serves as clear, user-initiated consent to receive an SMS solely for the purpose of logging in or creating an account. We do not send marketing or promotional messages from this number.

Messaging use case

  • Purpose: account authentication / login (one-time passcodes).
  • Message type: transactional, user-initiated OTP only.
  • No bulk messaging, campaigns, or marketing content.

Opt-out and help

Our SMS messages clearly inform users that they can reply "STOP" to opt out of further messages and "HELP" for assistance. Twilio's standard STOP/HELP handling is supported on this toll-free number.

Example OTP message

Your Mindo verification code is 123456. It expires in 10 minutes. Reply HELP for help, STOP to opt out.
